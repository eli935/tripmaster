### שינויים בטבלאות קיימות
```sql
-- shopping_items: תמיכה ב-dedup
ALTER TABLE shopping_items
  ADD COLUMN ingredient_normalized TEXT,
  ADD COLUMN source_meal_ids UUID[],
  ADD COLUMN auto_generated BOOLEAN DEFAULT true;

-- Backfill: נרמול הנתונים הקיימים
UPDATE shopping_items
SET ingredient_normalized = lower(trim(regexp_replace(ingredient, '[\u0591-\u05C7]', '', 'g')))
WHERE ingredient_normalized IS NULL;

ALTER TABLE shopping_items ALTER COLUMN ingredient_normalized SET NOT NULL;

CREATE UNIQUE INDEX idx_shopping_unique_per_trip
  ON shopping_items (trip_id, ingredient_normalized, unit)
  WHERE deleted_at IS NULL;

-- meals: מספר סועדים + תגיות מזוהות
ALTER TABLE meals
  ADD COLUMN diner_count INTEGER,
  ADD COLUMN detected_tags TEXT[];

CREATE INDEX idx_meals_detected_tags ON meals USING GIN (detected_tags);

-- trips: מודל דו-ממדי
ALTER TABLE trips
  ADD COLUMN location_type TEXT DEFAULT 'abroad'
    CHECK (location_type IN ('israel','abroad')),
  ADD COLUMN purpose TEXT DEFAULT 'friends'
    CHECK (purpose IN ('customer','friends','family')),
  ADD COLUMN billing_enabled BOOLEAN DEFAULT false,
  ADD COLUMN fee_amount NUMERIC(10,2),
  ADD COLUMN fee_currency TEXT DEFAULT 'ILS';

-- CHECK: billing_enabled=true דורש fee_amount>0
ALTER TABLE trips ADD CONSTRAINT trips_fee_check
  CHECK (
    (billing_enabled = false) OR
    (billing_enabled = true AND fee_amount IS NOT NULL AND fee_amount > 0)
  );
```

### טבלאות חדשות
```sql
CREATE TABLE ingredient_recipes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  dish_name TEXT NOT NULL UNIQUE,
  aliases TEXT[] DEFAULT '{}',
  ingredients JSONB NOT NULL,
  servings_base INTEGER DEFAULT 1,
  source TEXT CHECK (source IN ('seed','llm','user')) DEFAULT 'seed',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_ingredient_recipes_aliases ON ingredient_recipes USING GIN (aliases);
CREATE INDEX idx_ingredient_recipes_dish_lower ON ingredient_recipes (lower(dish_name));

CREATE TABLE recipes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  dish_name TEXT NOT NULL UNIQUE,
  instructions_md TEXT NOT NULL,
  prep_time_min INTEGER,
  cook_time_min INTEGER,
  difficulty TEXT CHECK (difficulty IN ('easy','medium','hard')),
  tips TEXT[],
  generated_by TEXT CHECK (generated_by IN ('seed','claude','user')),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_recipes_dish ON recipes (dish_name);
```

## 3 · תוצאה

- מיגרציה בודדת 006 — ~60 שורות SQL — transactional-safe (BEGIN/COMMIT).
- 3 אינדקסים חדשים: GIN על aliases ו-detected_tags, UNIQUE composite על shopping_items.
- CHECK constraints להבטיח שלמות של trip billing.
- Backfill של ~אלפי שורות shopping_items לנרמול — מוערך ב-<2 שניות על הסכמה הנוכחית.
- 0 BREAKING CHANGES — כל העמודות החדשות עם DEFAULT.

## 4 · הערכה עצמית (חובה כנות)

**מה עבד טוב:**
- מיגרציה אחת יוצרת הרבה, מפחיתה סיכון של שינויים חלקיים.
- כל העמודות החדשות עם DEFAULT — טיולים קיימים לא ייפגעו.
- אינדקס GIN על aliases יתן חיפוש O(1) יעיל עבור normalization.

**מה לא עבד / היה יכול להיות טוב יותר:**
- לא בדקתי מה קורה אם `shopping_items` קיים עם כפילויות היסטוריות שיפרו את ה-UNIQUE constraint. **חובה לבדוק לפני הרצה**: `SELECT trip_id, ingredient_normalized, unit, count(*) FROM shopping_items GROUP BY 1,2,3 HAVING count(*)>1`.
- regex הסרת ניקוד (`[\u0591-\u05C7]`) עובד רק אם הנתונים ב-UTF-8 תקני — לא בדקתי את הסט הקיים.
- לא חשבתי על RLS — המיגרציה לא מוסיפה policies. אבל RLS disabled כרגע, אז לא דחוף.
- לא בדקתי אם `expense_payers` או טבלאות אחרות צריכות שינוי עבור fee injection — השארתי את זה ל-financial-expert, אבל אולי אצטרך להוסיף עמודה.
- לא תכננתי rollback script — אם המיגרציה נכשלת באמצע, אין לי DROP.

**ציון עצמי:**
- ביצוע טכני: 8/10
- איכות התוצאה: 7/10 (חסר rollback)
- תקשורת עם סוכנים אחרים: 8/10
- עמידה בתקציב זמן: 9/10

## 5 · חסמים / פערי ידע

- **לא בדקתי** את ה-row count הנוכחי של `shopping_items` ו-`meals`. אם יש מיליון שורות, backfill יכול לקחת דקות ולנעול טבלה.
- **לא ידעתי** איך Supabase מגיב לטריגרים על realtime — פיצ'ר 1 ידרוש טריגר שיעדכן `shopping_items` מ-`meal_items`. אולי עדיף בצד האפליקציה.
- **פער** — האם `auto_generated` באמת נחוצה כעמודה בנפרד מ-`source_meal_ids IS NOT NULL`? רידנדנטיות אפשרית.
