# 🧠 ישיבת צוות דו-שבועית #1 — TripMaster

**תאריך:** 17/04/2026
**נושא:** ספרינט שיפורים לשבועיים הקרובים
**משתתפים:** tripmaster-manager + 6 סוכנים מקצועיים

---

## 📝 כל ההצעות (18 סה"כ)

### 🌍 destinations-expert

1. **Verified Kosher Data Layer** (10/4 = 2.5)
   החלפת המידע שנוצר ע"י AI במידע מאומת ידנית לכל יעד — הכשר, סטטוס פסח, שעות, טלפון. הוספת שדה `verified: true` + `last_verified_at`. התחלה עם 5 יעדים מובילים: מונטנגרו, רומא, אתונה, פראג, בודפשט.

2. **Shabbat & Chag Mode** (9/5 = 1.8)
   זיהוי אוטומטי של שבת/יו"ט לפי אזור הזמן של היעד. האפרה של אטרקציות שאינן במרחק הליכה, הצגת סטטוס עירוב, זמני הדלקת נרות, מרחק לבית כנסת הקרוב.

3. **Offline Trip Pack** (8/6 = 1.33)
   יצירת PDF + PWA cache של המידע המלא על היעד (חב"ד, מפת כשרות, אטרקציות, מספרי חירום, ביטויים בסיסיים בעברית) — מקטין תלות ב-roaming בחו"ל.

### 💰 financial-expert

1. **דשבורד "מי חייב למי" חי + כפתור העברה** (10/5 = 2.0)
   רכיב שמציג את פלט minimize-transfers כקלפים ("עקשטיין → רוזנפלד ₪3,671") עם כפתורי "סמן כשולם" + deep-link ל-Bit/PayBox. מוריד 3 שיחות וואטסאפ לקליק אחד.

2. **נעילת שער חליפין לכל הוצאה** (9/4 = 2.25)
   שערי Frankfurter חיים — אם EUR/ILS משתנה, היתרות זזות והלקוחות מאבדים אמון. הוספת `fx_rate` ו-`fx_locked_at` בטבלת expenses.

3. **אשף הוצאה מרובת-משלמים ב-3 צעדים** (8/6 = 1.33)
   החלפת הטופס הנוכחי ב-wizard: סכום+מטבע ← מי שילם כמה ← איך מחלקים. חותך זמן הזנה מ-60 ל-15 שניות.

### 📦 logistics-expert

1. **שיוך פריטים למשפחות — Who Brings What** (9/4 = 2.25)
   עמודת `assigned_family_id` בטבלת equipment. UI עם dropdown ליד כל פריט + תצוגת "הרשימה שלי" מסוננת לכל משפחה.

2. **התראות חכמות לפריטים קריטיים** (8/3 = 2.67) ⭐
   סימון `is_critical` (גז, תרופות, מסמכים, מפתחות) + באדג' אדום "3 פריטים קריטיים עדיין pending" 48 שעות לפני היציאה.

3. **למידה מטיולים קודמים — Smart Quantities** (7/5 = 1.4)
   טבלת `trip_feedback` עם `actual_used` לכל פריט. אלגוריתם שמתקן את `qty_per_person` לפי ממוצע משוקלל.

### 📖 halacha-expert

1. **זמני הדלקת נרות + זמני הלכה לכל יעד** (10/4 = 2.5) ⭐
   שאילתת Hebcal API למיקום+תאריך. תצוגה בכרטיסי ערב שבת/חג עם בורר 18/40 דקות לפי מנהג.

2. **אזהרת בישול/חם בשבת וחג** (9/3 = 3.0) ⭐⭐
   באנר אדום על ארוחות בשבת/חג. אזהרה על בישול עכו"ם, דרישת פלטה. trigger אוטומטי כשהיעד הוא מלון/מסעדה לא `mehadrin_kitchen=true`.

3. **דגל "נוהג ישראל/חו"ל"** (8/2 = 4.0) ⭐⭐⭐
   הגדרת `user.custom: "israel" | "chutz"`. בנסיעה לחו"ל ביו"ט — הוספה אוטומטית של יו"ט שני עם הערה הלכתית.

### 🎨 ui-designer

1. **Hero Imagery + כרטיסי יעד** (9/4 = 2.25)
   תמונות full-bleed של יעדים (Unsplash API לפי שם עיר) עם gradient overlay, parallax scroll, Ken Burns zoom בהובר. Blurhash placeholders.

2. **Micro-interactions + Haptic Feedback** (8/5 = 1.6)
   spring-scale על לחיצה (0.96), confetti בהשלמת משימה, skeleton shimmer, swipe-to-delete, pull-to-refresh, `navigator.vibrate()` במובייל.

3. **Bottom Nav + Gesture Tabs** (9/6 = 1.5)
   החלפת 11 הטאבים העליונים ב-floating glass bottom-nav (5 אייקונים + "עוד"). swipeable tab views עם snap scrolling. ⚠️ שובר הרגלים.

### 🗄️ database-expert

1. **הפעלה מחדש של RLS עם policies נכונים** (10/5 = 2.0) 🔐
   סגירת פרצת האבטחה הקריטית. כתיבת policies מבוססי `auth.uid()` עם `SECURITY DEFINER function` (`is_trip_member(trip_id)`) כדי למנוע רקורסיה. הפעלה הדרגתית על 18 טבלאות עם בדיקות.

2. **גיבויים אוטומטיים יומיים + retention 30 יום** (9/3 = 3.0) ⭐⭐
   GitHub Action יומי שמריץ pg_dump דרך Supabase, דוחס, מעלה ל-S3/R2. בדיקת-שחזור שבועית.

3. **Partition + retention ל-audit_log ו-destinations_cache** (7/4 = 1.75)
   המרת `audit_log` ל-partitioned by month + `pg_cron` שמוחק חודשים מעל 90 יום. TTL ל-`destinations_cache` (מחיקה ללא hit מעל 60 יום).

## 🎯 5 ההצעות המובילות (לאחר חשיבה משותפת)

כאשר ממזגים אבטחה + הלכה + אמון + לוגיסטיקה + עיצוב — אלה 5 ההצעות שיזיזו הכי הרבה:

### 1. 🔐 RLS Re-enablement
**קטגוריה:** אבטחה קריטית
**ציון:** 10/5
**למה מוביל:** פרצה קיימת — כל משתמש יכול לקרוא כל טבלה. חובה לפני הוספת משתמשים חדשים. גם ציון מתמטי בינוני, אבל סיכון עסקי = חובה.
**בעלים:** database-expert
**זמן:** 2 ימים

### 2. 🕯️ זמני הלכה + הדלקת נרות לכל יעד
**קטגוריה:** מקצועיות חרדית
**ציון:** 10/4 = 2.5
**למה מוביל:** כל משפחה חרדית ישר תשאל "מתי כניסת שבת בבודווה?" — היום אין תשובה באפליקציה. Hebcal API חינמי. פיצ'ר "הצהרה" שאנחנו יודעים מה אנחנו עושים.
**בעלים:** halacha-expert + destinations-expert
**זמן:** 2-3 ימים

### 3. 💰 דשבורד "מי חייב למי" + נעילת שער
**קטגוריה:** חשבונות
**ציון:** 10/5 + 9/4 = שילוב
**למה מוביל:** זה **הכאב המיידי** שלך ושל עקשטיין כרגע. במקום לפתוח וואטסאפ ולחשב — רואים כרטיס עם הסכום וכפתור Bit.
**בעלים:** financial-expert + ui-designer
**זמן:** 2 ימים

### 4. 🔔 פריטים קריטיים + שיוך משפחות
**קטגוריה:** לוגיסטיקה
**ציון:** 8/3 + 9/4 = שילוב
**למה מוביל:** משלב 2 כאבים — "שכחתי את התרופות!" + "מי מביא את הסוכה?". שתי הצעות שמשלימות זו את זו.
**בעלים:** logistics-expert
**זמן:** 2-3 ימים

### 5. 🎨 Hero Imagery + Micro-interactions
**קטגוריה:** עיצוב ו-WOW factor
**ציון:** 9/4 + 8/5 = שילוב
**למה מוביל:** זה השינוי שיגרום לחבר שלך שראה את האפליקציה לומר "וואו, זה יפה". חיוני אם אתה חושב על מסחור.
**בעלים:** ui-designer
**זמן:** 3 ימים

## ⚠️ סיכונים וחששות

1. **RLS** עלול לשבור queries קיימים (ראינו את זה בעבר). נדרש regression testing מקיף אחרי כל policy.
2. **Hebcal API** — rate limits. נדרש cache חזק (לפחות 24 שעות per location/date).
3. **Bottom Nav** (הצעה 3 של UI) — שובר הרגלים. ממליצים לדחות עד שהמוצר בוגר יותר.
4. **Critical items** — צריך להחליט מי קובע מה "קריטי". הצעה: admin מסמן, משתמשים יכולים לבקש לסמן.
