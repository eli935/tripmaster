## 0 · Framing

TripMaster is not a SaaS dashboard. It is a **tool for religious Jewish families planning sacred time together** — multi-family trips woven around zmanim, chag, and Shabbat. The current v8.6 UI is "premium dark" but generic: purple→blue gradients that would fit any fintech, glass cards that feel Linear-ish, no Jewish soul.

This vision trades "premium dark SaaS" for **editorial travel magazine x Jerusalem stone x midnight aurora** — a design that could only exist for *this* product, for *this* audience. Grounded in the frontend-design skill's mandate: "What makes this UNFORGETTABLE? What's the one thing someone will remember?" (`frontend-design/SKILL.md`, Design Thinking section).

## 1 · Localization & UX Bug Fixes (Section 1 — ship before aesthetics)

### 1.1 Payer UUID leak — `app/trip/[id]/trip-overview.tsx:795`
```tsx
{(exp as any).payer?.full_name} · {EXPENSE_CATEGORIES[exp.category as ExpenseCategory] || exp.category}
```
Problem: when `payer` relation is not joined (stale realtime payload, or row inserted without refetch), renders `undefined` — and in other places the raw UUID leaks. `profileNames` is already built at line 748 but not used here.

### 1.4 Tab navigation — the flagship UX bug

**Current** (`trip-overview.tsx:201-225`): 12 pill tabs in `flex gap-2 overflow-x-auto scrollbar-hide`. On desktop the user cannot see there is more content to the right — the scrollbar is hidden. Arrow-key scrolling the tab strip is undiscoverable. "מנהל" and "הגדרות" are effectively invisible.

**Rejected options:**
- *Visible scrollbar*: cheap, un-premium. Violates ui-ux-pro-max §5 `scroll-behavior` (avoid nested scroll regions competing with main scroll).
- *Two-row wrap*: breaks rhythm, makes the page feel like an admin console.
- *Single dropdown*: hides hierarchy — a Jewish family should see "יעד · סקירה · ארוחות · הוצאות" at a glance.

**Proposed — "Command Deck":**
1. **Primary row (always visible, never scrolls)** — the 5 most-used tabs, sized by usage data:
   `סקירה · ארוחות · הוצאות · צ׳אט · יעד`
   These get the gradient-blue active pill treatment currently in use.
2. **"עוד" overflow pill (last position)** — opens a **bottom sheet on mobile, Radix Popover on desktop**, with grouped sections:
   - **תכנון**: ציוד · קניות · לקחים · קבצים
   - **סיכום**: סיכום מסע
   - **ניהול**: מנהל · הגדרות (only for `isAdmin`)
3. **Cmd-K palette** — `⌘K` / `Ctrl+K` opens a search palette (shadcn `Command`) with fuzzy search over all tabs + quick actions ("הוסף הוצאה", "הוסף ארוחה", "שלח WhatsApp"). This is the single biggest power-user win — inspired by Linear (Emil Kowalski).
4. **Animated underline (`layoutId="active-tab"`)** moves between primary pills using Motion's shared-layout animation — continuity when switching.

This is UX + signature motion + power-user feature in one fix. Grounded in `ui-ux-pro-max/SKILL.md` §9 `overflow-menu` ("When actions exceed available space, use overflow/more menu instead of cramming") and `nav-hierarchy` ("Primary nav vs secondary nav must be clearly separated").

## 2 · Aesthetic Direction

### The choice: **"Editorial Jerusalem Gold on Midnight OLED"**

Chosen from the ui-ux-pro-max style library intersection of: *editorial/magazine*, *dark mode OLED*, *luxury/refined*, with a **single dominant accent**: warm Jerusalem-stone gold (not the current purple/pink pastel gradient-gold `#f6d365→#fda085` which reads as sunset-smoothie, but a proper editorial gold: `#D4A960` warm, `#F4E4BC` highlight, `#8B6F3A` shadow — colors you'd see on the stone of the Kotel at dusk).

**Why it fits:**
- **Religious Jewish family audience.** Gold is halachically neutral but culturally resonant (menorah, kiddush, Jerusalem at sunset). Purple fintech gradients are not.
- **"Premium travel app"** ≠ "premium SaaS". This audience reads Condé Nast Traveller, not TechCrunch. Editorial typography and generous whitespace speak their language.
- **Single dominant accent** outperforms "timid, evenly-distributed palettes" (`frontend-design/SKILL.md`). Right now we have 6 gradient classes all fighting — blue AND green AND orange AND gold AND purple AND teal. That's not a palette, that's a color swatch book. Kill 4 of them.

### Typography — this is where the brand lives

**Current:** Rubik for everything. Rubik is functional Hebrew/Latin but not distinctive — `frontend-design/SKILL.md` explicitly warns: *"Avoid generic fonts like Inter/Arial; opt for distinctive choices that elevate aesthetics."*

**Proposed pairing:**
- **Display (headlines, numbers, hero):** `Frank Ruhl Libre` (Google Fonts, Hebrew + Latin, serif with Jerusalem roots — named after the Frank-Ruhl Hebrew typeface used in Israeli book publishing for decades). Weight 500 for body headlines, 900 for hero numerals.
- **Body + UI:** keep `Rubik` — it's readable, Hebrew-native, shipped. Don't break what works.
- **Numerals (balance dashboard, counts, prices):** `Fraunces` tabular figures, variable font, for that editorial-magazine feel. Use `font-variant-numeric: tabular-nums` religiously on data.

The display serif on trip names + big balance numbers is the single fastest way to stop looking like a SaaS.

### Color system — replace `globals.css` theme

```css
:root {
  /* Base — deeper, more OLED */
  --background: oklch(0.08 0.01 260);   /* was 0.13 — push to near-black */
  --card:       oklch(0.12 0.01 260);
  --foreground: oklch(0.97 0 0);
  --muted-foreground: oklch(0.62 0 0);

  /* The signature gold — warm Jerusalem stone */
  --gold-50:  #FAF3E2;
  --gold-200: #F4E4BC;
  --gold-500: #D4A960;  /* PRIMARY ACCENT — replaces blue primary */
  --gold-700: #8B6F3A;
  --gold-900: #3D2E15;

  /* Semantic support */
  --sapphire: #1B3A6B;   /* rare cold counter-accent for chat / info */
  --olive:    #5A6B3C;   /* success, positive balance */
  --rose:     #8B2E3C;   /* debt, warning — not bright red */
}
```

### What stays
- Dark base (deepened)
- Glass morphism (refined — see §4)
- Framer-motion architecture
- The gold balance hero from v8.6 (now the color's promoted from accent to primary)
- RTL, Rubik body
- `ease-out-expo` `[0.16, 1, 0.3, 1]` — it's correct

### What dies
- `gradient-purple`, `gradient-orange`, `gradient-teal` — **deleted**. Orphan gradients from 2022 dashboards.
- `gradient-blue` (`#667eea→#764ba2`) — **deleted**. The purple fintech cliché `frontend-design/SKILL.md` names as a "generic AI-generated aesthetic" to avoid. Replace with `gradient-gold-editorial` (`#F4E4BC→#D4A960→#8B6F3A` — three stops, warm).
- `text-glow` blue halo — replace with a warm gold version `rgba(212, 169, 96, 0.25)`.
- `animate-float` — remove from production UI. It's a toy. Keep only on empty-state illustrations.
- `scrollbar-hide` on the tab strip (see §1.4).
- Rubik as heading font (demoted to body-only).

## 3 · Motion Philosophy

### Lead designer: **Jakub Krehel** (production polish) — with Emil Kowalski for speed discipline and Jhey Tompkins for the ONE signature moment (see §7).

Per `design-motion-principles/SKILL.md` context mapping: mobile app + e-commerce hybrid + family/consumer → **Jakub primary, Emil secondary, Jhey selective**.

### Rules
- **Enter recipe (universal):**
  ```tsx
  initial={{ opacity: 0, y: 8, filter: "blur(4px)" }}
  animate={{ opacity: 1, y: 0, filter: "blur(0px)" }}
  transition={{ type: "spring", duration: 0.45, bounce: 0 }}
  ```
  The blur entry is Jakub's signature — feels like focus pulling into place.
- **Exit is subtler than enter** — `y: 4`, duration `0.25`, same blur.
- **Hover on interactive cards**: `scale: 1.015` (not 1.05 — we're not a toy), `y: -2`, 200ms. Gold border gains opacity from `0.12` → `0.35`.
- **Tap**: `scale: 0.98`, 120ms.
- **Page-level stagger**: children 40ms apart, max 6 staggered items (cap it or it feels slow). `stagger-sequence` rule from ui-ux-pro-max §7.
- **Total page entrance budget**: 800ms. Nothing animates after that unless user interacts.
- **Exit-faster-than-enter** (ui-ux-pro-max §7) — exits ~60% of enter duration.
- **`prefers-reduced-motion`** — MANDATORY. Wrap global `useReducedMotion()` hook in a layout provider, disable blur + spring, keep opacity. Not optional.

### What animates
- Tab switches (already done, refine to layout-id underline)
- Balance hero count-up (already done, keep)
- Card entrances (new — blur-in staggered)
- Destination hero parallax on scroll (new — see §4)
- The ONE signature moment (see §7)

### What stays static
- Body text
- Form inputs
- Navigation chrome after first load
- Tables and lists after initial render

> Jakub's golden rule: *"The best animation is that which goes unnoticed."* Exception: §7.

## 5 · New Global Patterns

### 5.1 Signature micro-interactions
- **Gold shimmer sweep** (Jhey, restrained): new items (freshly added expense, just-received chat) pulse a 1.2s diagonal gold-highlight gradient sweep on first render, then never again. Uses `motion-framer` keyframes array.
- **Magnetic buttons** on primary CTAs ("הוסף הוצאה", "שלח WhatsApp"): cursor within 40px radius pulls button 2-3px toward pointer. Disabled on touch devices + reduced-motion.
- **Number morph** on balance changes (`motion` `animate={{ x }}` on a `useMotionValue`, interpolated into a displayed string). Currently count-up runs once; this replays on value change.

### 5.2 Scroll effects
- **Destination hero parallax** (§4.4) — `useScroll` + `useTransform`.
- **Sticky tab bar** with gold bottom-border that fades in only after scrolling past the trip header (`whileInView` + `viewport={{ margin: '-80px' }}`).
- **No parallax on lists, cards, chat** — parallax costs attention; spend it on the hero only.

### 5.3 Loading states
Replace all `<Loader2>` spinners with **skeleton shimmer** in gold tones. Per ui-ux-pro-max §3 `progressive-loading` — skeletons for anything over 300ms. Skeleton uses `gradient-shimmer` with gold, not blue.

### 5.4 Empty states
Current: plain `text-muted-foreground` centered. Proposed: **illustrated empty states** per section — hand-drawn SVG in gold on midnight (compass for trips, stove for meals, scroll for lessons, coins for expenses). Beneath: one-line Hebrew prompt in `Frank Ruhl Libre italic`, then a single CTA.

### 5.5 Focus rings
Replace default blue ring with `ring-2 ring-gold-500/60 ring-offset-2 ring-offset-background`. Accessibility non-negotiable (ui-ux-pro-max §1 `focus-states`).

## 7 · The ONE Unforgettable Thing

Per `frontend-design/SKILL.md`: *"What makes this UNFORGETTABLE? What's the one thing someone will remember?"*

### **"Havdalah Transition"** — Shabbat mode across the entire app

Every Friday at local candle-lighting time (we already have zmanim data), the entire app **fades into Shabbat mode** with a one-time, 2.5-second transition:

1. The ambient gold of the app **warms and dims** — accent color shifts `#D4A960 → #B8874A`, background deepens by 8%, as if the sun is setting.
2. A **single candle flame SVG** (Jhey-built, CSS `@property` + `linear()` flicker) appears bottom-left corner for 4 seconds, then settles to a static glow.
3. All non-essential animations **pause**. Realtime chat goes read-only with a Hebrew explanation: *"שבת שלום — הצ׳אט יחזור במוצאי שבת"*.
4. Mutations are queued locally, flushed at havdalah.
5. At havdalah (also from zmanim), the reverse: a **brief spice-box shimmer** (cinnamon color sweeps across the viewport once, 1.8s), colors return to full saturation, chat reconnects, and a small toast: *"שבוע טוב"*.

This is:
- **Halachically respectful** (the app actively honors Shabbat)
- **Technically ambitious** (zmanim-driven, app-wide state, graceful degradation)
- **Emotionally unforgettable** (no other travel app does this; your religious users will tell their friends)
- **Brand-defining** (ties the editorial gold aesthetic directly to the audience's sacred time)
- **Bounded** (triggers twice a week, never feels gimmicky)

Nothing else in this document is as important. If Day 12 only ships this, v8.7 is a success.
